#  Docker 介紹

Docker 是一個開源的容器化平台，能夠將 `應用程式與其執行環境` 打包成 <u> 輕量且可攜帶</u> 的容器（Container）。這些容器在任何有安裝 Docker 的系統上，都能一致且快速地運行，避免「環境不一致」的問題。

Docker 利用的是 Linux 核心的 namespace 和 cgroups 技術。

---

## 為什麼要使用 Docker？

- **環境一致性**  
  將應用及所有依賴包裝在容器中，不論開發、測試或生產環境，都能保證相同的運行條件，減少「在我機器上可以運行」的狀況。

- **快速部署與啟動**  
  容器啟動速度快，不需要啟動完整作業系統，節省開發和部署時間。

- **資源利用率高**  
  容器輕量，共享宿主核心，消耗較少資源，能在同一台機器上運行更多服務。

- **彈性與擴展性**  
  容器易於複製和擴展，方便建構微服務架構及自動化部署流水線 (CI/CD)。

- **簡化維護與管理**  
  利用映像管理、版本控制，方便回溯及更新應用環境。



## Docker 和 VM 

在Docker 尚未推出時，是透過建立 VM (Virtual Machine) 來解決「環境不一致」的問題。

VM 就是一台完整功能的電腦，只不過是虛擬的。因為這台虛擬電腦是由一台實體電腦所模擬出來。

Q : 為甚麼要由一台實體電腦模擬一台虛擬電腦出來?  

A : 當你只有一台電腦時， 卻同時需要 Windows 的環境 和 Linux 的環境的，甚至更多不同環境時候。

如果不用VM，實現的方法就是當你想要用Linux的時候，把電腦重灌成Windows；當你想用Windows的時候把電腦重灌成Linux。顯然是不可能，因此VM誕生了。想在自己的電腦中建立VM，有兩個條件。第一個是確認自己的CPU 是不是有支援虛擬化技術 ，再來就是在自己的電腦上安裝Hypervisor。ex: VirtualBox。這樣你的電腦就具備打造 VM 的 能力了。



| 特性           | Docker 容器                     | 虛擬機（VM）                        |
|--------------|----------------------------|--------------------------------|
| 是否需要硬體虛擬化支援 | 不需要                         | 需要       |
| 隔離方式         | 利用 Linux kernel namespace 和 cgroups | 利用 Hypervisor 模擬完整硬體及 OS     |
| 資源消耗         | 輕量，共用核心                   | 重，需運行完整 OS                   |
| 作業系統彈性   |  必須與宿主機核心相容（通常 Linux）  |可跑不同作業系統（Linux、Windows等）|

---


如何選擇?  

- VM : 完整、獨立且強隔離的系統環境，或運行不同作業系統
- Docker : 快速啟動、輕量、易於擴展及持續交付，特別是微服務架構


## 總結 : 

Docker 是現代軟體開發與運維的重要工具，幫助團隊加速產品交付，提升系統穩定性與效率。
